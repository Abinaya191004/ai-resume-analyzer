## Local Development

### 1. Start Backend Server (Port 3000)
```bash
cd Backend
npm install
npm start
```

The Backend will:
- Start Express server on http://localhost:3000
- Serve the Frontend from `/Frontend/dist` (production build)
- Expose `/api/analyze` endpoint for resume analysis

### 2. Frontend Access
Once Backend is running, access the application at:
```
http://localhost:3000
```

The Frontend is already built in `/Frontend/dist` and will be served by the Backend.

## Production Deployment

### Option A: Deploy to Netlify (Frontend Only)
1. Create a new site on Netlify
2. Connect to your GitHub repository
3. Set Build Command: `cd Frontend && npm run build`
4. Set Publish Directory: `Frontend/dist`
5. Add environment variables if needed
6. Deploy

**Note:** You'll also need to deploy the Backend separately (see Option B)

### Option B: Deploy Backend + Frontend (Recommended)
Deploy the entire project to a Node.js hosting service (Render, Heroku, Railway, etc.):

1. Ensure `Frontend/dist` is built (run `npm run build` in Frontend folder)
2. Push to GitHub
3. Connect repository to hosting platform
4. Set Start Command: `npm start` (from Backend directory) OR `cd Backend && npm start`
5. Set Environment Variables:
   - `PORT=3000` (or hosting platform's default)
   - `OPENROUTER_API_KEY=your_api_key_here`
   - `MONGODB_URI=your_mongodb_connection_string` (optional)

## API Endpoint

### POST `/api/analyze`
Analyze a resume against a job description.

**Request:**
```json
{
  "resumeText": "Your resume content here...",
  "jobDescription": "Job description content here..."
}
```

**Response:**
```json
{
  "overallScore": 85,
  "scoreBreakdown": {
    "skills": 25,
    "experience": 20,

  "jdSkills": ["JavaScript", "Node.js", "React", "MongoDB"],
  "jdMatch": 75,
  "missingSkills": ["MongoDB"]
}
```

## Troubleshooting

### Issue: 404 errors when accessing frontend
**Solution:** Ensure `/Frontend/dist` folder exists and Backend is serving static files
```bash
cd Frontend && npm run build
```

### Issue: API calls failing with 500 error
**Solution:** Check OPENROUTER_API_KEY is set in .env file
```bash
OPENROUTER_API_KEY=sk-or-v1-your-actual-key
```

### Issue: Port already in use
**Solution:** Change PORT in .env or kill process using port 3000
```bash
PORT=8080
```
